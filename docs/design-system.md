# Design System

## Project Vision

Build a modern, trustworthy, simple and premium website for a local business providing insurance, documentation and financial assistance.

The website should be easy enough for first-time internet users while still looking professional.

---

# Brand Personality

- Professional
- Trustworthy
- Premium (not luxurious)
- Clean
- Helpful
- Modern
- Minimal but informative

---

# Target Audience

Primary Audience

- People from Pune and Mumbai
- People looking for insurance
- People needing government documentation
- Families
- Small business owners
- Users with limited technical knowledge

---

# Design Principles

1. Clarity over creativity.

2. Trust over decoration.

3. Every section should have one primary purpose.

4. Information should be easy to scan.

5. Important actions (Call & WhatsApp) should always be accessible.

6. Avoid clutter.

7. Use real information instead of marketing buzzwords.

---

# Layout

Maximum Content Width

- 1280px

Section Padding

- Comfortable spacing
- No excessive whitespace

Navigation

- Sticky
- White background
- Thin bottom border

---

# Components

Navbar

- Sticky
- Logo
- Navigation
- Call Button
- WhatsApp Button

Hero

- Large heading
- Short description
- Two CTA buttons
- Trust indicators

Cards

- Slightly rounded corners
- Thin border
- Subtle shadow
- Hover effect

Buttons

Primary

- Filled

Secondary

- Outlined

# Images

Use

- Real office photos
- Real owner photos
- Professional icons

Avoid

- Generic AI illustrations
- Cartoon graphics

---

# Animations

- Subtle only
- Fast
- Functional
- No unnecessary motion

---

# Homepage Structure

1. Hero
2. Services
3. Process
4. Why Choose Us
5. Required Documents
6. About
7. Testimonials
8. FAQ
9. Contact
10. Footer

---

# Mobile First

Every section must work perfectly on mobile before desktop.

---

# Accessibility

- Semantic HTML
- Keyboard navigation
- Good color contrast
- Large touch targets

---

# Performance

Avoid unnecessary libraries.

Prefer built-in Next.js features.

Keep components modular.

---

# Engineering Rules

- One component = One responsibility.
- Reusable before duplicated.
- Keep files small and readable.
- Prefer data-driven UI.
- No hardcoded business information inside components.

# Things We Will Not Do

- No glassmorphism.
- No excessive gradients.
- No auto-playing videos.
- No unnecessary animations.
- No giant hero sections with empty whitespace.
- No walls of text.
- No generic AI-style startup layouts.
- No inconsistent spacing.
- No duplicate UI components.
