# AI_CONTEXT.md

## Project Overview

Project Name: **MS Insurance & Documentation**

This is a production-grade full-stack business website for a local consultancy in India.

The website represents a real business and should feel trustworthy, professional, modern, and easy to understand.

The goal is to help customers quickly understand available services and contact the business with confidence.

---

# Target Audience

Primary audience:

* People from Pune and Mumbai
* Families
* Working professionals
* Small business owners
* People looking for insurance
* People requiring government documentation
* Users with limited technical knowledge

The UI should be understandable even for someone who is not comfortable using technology.

# Project Goals

The website should:

* Build trust.
* Reduce customer confusion.
* Explain processes clearly.
* Increase inquiries.
* Encourage WhatsApp contact.
* Allow future expansion without major redesign.

---

# Design Philosophy

The website should feel:

* Professional
* Trustworthy
* Premium (not luxurious)
* Clean
* Modern
* Helpful
* Simple
* Timeless

Avoid anything that looks like a generic AI-generated startup landing page.

---

# Design Principles

* Clarity over creativity.
* Trust over decoration.
* Information should be easy to scan.
* Every section should have one primary purpose.
* Use whitespace carefully.
* Do not create clutter.
* Use real information instead of marketing buzzwords.
* Mobile experience is equally important as desktop.

---

# Things To Avoid

Do NOT use:

* Glassmorphism
* Excessive gradients
* Giant empty hero sections
* Cartoon illustrations
* Overly rounded "bubble" UI
* Heavy animations
* Fancy scrolling effects
* Unnecessary dependencies
* Duplicate components

---

# Coding Standards

Always:

* Use semantic HTML.
* Write responsive components.
* Keep components focused on one responsibility.
* Keep code readable.
* Prefer reusable solutions.
* Avoid hardcoded business information.
* Prefer mapping over repeated JSX.
* Follow accessibility best practices.
* Use TypeScript properly.

---

# Engineering Philosophy

Treat AI as a junior developer.

Do not invent features that were not requested.

Do not over-engineer simple components.

Prefer maintainability over cleverness.

Generate production-quality code that another engineer can easily understand.

Always optimize for clarity, scalability, and long-term maintainability.
